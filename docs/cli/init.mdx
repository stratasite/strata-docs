# strata init

Initialize a new Strata project.

## Synopsis

```bash
strata init [PROJECT_NAME] [OPTIONS]
```

## Description

Creates a new Strata project with the following structure:

```
PROJECT_NAME/
├── project.yml          # Project configuration
├── datasources.yml      # Datasource definitions
├── .strata              # Local secrets (gitignored)
├── models/              # Semantic model files
├── migrations/          # Migration files
└── tests/               # Test definitions
```

## Arguments

### PROJECT_NAME

Optional project name. Required unless using `--source`.

## Options

### -d, --datasource ADAPTER

Add datasource adapters during initialization. Can be repeated.

```bash
strata init my-project -d postgres -d snowflake
```

**Supported adapters:**
- `postgres`
- `snowflake`
- `mysql`
- `sqlserver`
- `athena`
- `trino`
- `duckdb`
- `druid`

### -s, --source URL

Clone existing project from URL.

```bash
strata init --source https://github.com/example/strata-project.git
```

### -a, --api-key KEY

API key for cloning from source. Required with `--source`.

```bash
strata init --source URL --api-key YOUR_API_KEY
```

## Examples

### Basic Initialization

```bash
strata init my-project
```

### With Datasources

```bash
strata init my-project -d postgres -d snowflake
```

### Clone from Source

```bash
strata init --source https://github.com/example/project.git --api-key KEY
```

## Project Configuration

After initialization, edit `project.yml`:

```yaml
name: My Project
description: Project description
server: https://strata.example.com
production_branch: main
git: https://github.com/example/project.git
```
